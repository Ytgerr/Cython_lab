Key Findings

  • Cython untyped (~2x): Same Python code compiled to C. Variables are still PyObject* — boxing overhead remains. Only the interpreter dispatch loop is eliminated.
  • Cython typed (50–100x for Monte Carlo): cdef int i, cdef double x, y put variables on the C stack. with nogil: releases the GIL. libc.stdlib rand() replaces Python’s random.random(). Result: pure C loop with zero Python overhead.
  • Monte Carlo is the best boxing demo: Every iteration touches only scalar variables. In Python each x, y, inside is a PyFloatObject (24 bytes, heap-allocated). In typed Cython they are double / int on the stack — no malloc, no GIL.
  • Memory: Python list of lists stores each float as PyFloatObject (24 bytes). Cython typed uses NumPy arrays — 8 bytes per element. ~4x memory savings for 128x128 matrix.
  • Cython typed ≈ C: The generated .c file for typed code is essentially hand-written C. Difference vs pure C is <5%.
