Key Findings

  • Cython untyped (~1.8x): Same Python code compiled to C. Variables are still PyObject* — boxing overhead remains. Only the interpreter dispatch loop is eliminated.
  • Cython typed (~62x for matmul): cdef int i, j, p and cdef double s put variables on the C stack. double[:, ::1] typed memoryviews give direct pointer arithmetic. with nogil: releases the GIL. Result: pure C loop with no Python overhead.
  • Memory: Python list of lists stores each float as PyFloatObject (24 bytes). Cython typed uses NumPy arrays — 8 bytes per element. 4x memory savings for 128x128 matrix.
  • Scaling: Speedup is consistent across matrix sizes — the bottleneck is the inner loop, not Python call overhead.
  • Cython typed ≈ C: The generated .c file for typed code is essentially hand-written C. Difference vs pure C is <5%.
